# Расширенный API + поведенческие заметки

В SKILL.md есть короткая таблица методов. Здесь — детали, которые не нужны при каждой генерации, но всплывают на edge cases.

## TitleConfig — содержимое титульного листа

```
TitleConfig(
    # --- Обязательные ---
    work_type: str,         # «Лабораторная работа», «Курсовая работа», «ВКР», «Отчёт по практике»
    topic: str,             # тема работы, БЕЗ кавычек (ГОСТ запрещает кавычки вокруг темы)
    student_name: str,      # «Фамилия И.О.»
    student_group: str,     # «P3XXX», «20.Б10-мм»
    year: str,              # «2026»

    # --- Опциональные ---
    work_number: str = "",     # «№1», «№3»
    variant: str = "",         # для лаб с вариантами
    teacher_name: str = "",    # «Фамилия И.О.»
    teacher_label: str = "Проверил",   # для ВКР/курсовых: "Руководитель"
    teacher_degree: str = "",          # «к.т.н.», «д.ф.-м.н.»
    teacher_position: str = "",        # «доцент», «профессор»

    # --- Переопределение полей профиля ---
    city: str = "",
    ministry: str = "",
    university_full: str = "",
    university_short: str = "",
    faculty: str = "",
)
```

Поля университета (`city`, `ministry`, `university_full`, `university_short`, `faculty`) **по умолчанию пустые** и наследуют значения из активного `UniversityProfile`. Заполнять в `TitleConfig` стоит только если на конкретном документе нужно отличие от профиля.

## Поведенческие заметки

### TOC это поле Word

`r.toc()` вставляет настоящее поле Word (FIELD `TOC \\o "1-3" \\h \\z \\u`), не статический текст. Сразу после открытия файла оно пустое (placeholder). Пользователь должен:
1. Открыть `.docx` в Word (не LibreOffice, не PDF preview).
2. ПКМ по полю оглавления → «Обновить поле».

**Скажи это пользователю в ответе** — иначе он подумает, что оглавление сломано.

### Первый h1 не делает двойной разрыв

После `r.toc()` (который сам вызывает `page_break()`) и после титульной страницы первый `r.h1(...)` не вставляет лишний разрыв. Логика управляется флагом `_just_broke_page` внутри. Не пытайся добавлять `r.page_break()` руками перед первым h1.

### Списки рестартуют с 1

Каждый `r.numbered(...)` начинает новый нумерованный список с пункта 1 — независимо от предыдущих. То же для `r.bullet(...)`. Чтобы продолжить один список, передавай все пункты одним вызовом:

```python
# Неправильно — два списка, каждый начинается с 1:
r.numbered(["Шаг 1"])
r.text("Промежуточный текст.")
r.numbered(["Шаг 2"])  # тоже стартует с 1

# Правильно — один список:
r.numbered(["Шаг 1", "Шаг 2"])
```

### Без кавычек вокруг темы

ГОСТ 7.32 запрещает кавычки вокруг темы работы. Библиотека передаёт `topic` как есть — сам не оборачивай:

```python
# Неправильно:
TitleConfig(topic='«Численные методы»', ...)
TitleConfig(topic='"Численные методы"', ...)

# Правильно:
TitleConfig(topic='Численные методы', ...)
```

### Шрифты и поля не трогать вручную

Не настраивай размеры шрифта / поля страницы / межстрочный интервал через `r.doc` (raw python-docx). Если нужны другие параметры, собери `UniversityProfile` (см. `references/profiles.md`). Драйвер `r.doc` это аварийный выход для совсем экзотических случаев типа альбомной ориентации страницы.

### Placeholder-имена в примерах

В качестве имён студента/преподавателя всегда пиши «Фамилия И.О.», а не реальные. Пользователь сам подставит свои. Это про:
- `student_name`
- `teacher_name`

### Авто-подписи рисунков и таблиц

Префиксы «Рисунок N — ...» и «Таблица N — ...» формирует код. Тире в этих префиксах — обязательное по ГОСТ, библиотека его ставит сама. **Твой `caption` в эти префиксы попадает уже после санитайзера**: длинные тире в нём становятся запятыми. То есть когда ты пишешь `r.figure(path, "Схема сети")`, в документ попадает «Рисунок 3 — Схема сети» — это правильно, тире здесь от библиотеки, не от тебя.

### Картинки кламяются по ширине

`r.figure(image_path, caption, width_cm=None)` всегда ограничивает ширину картинки печатной областью (A4 минус поля профиля). Если `width_cm=None` — используется натуральный размер картинки, но клампится. Если `width_cm` задан — он уважается, но тоже клампится. Это защита от вылета крупных скриншотов за поля.

### Санитайзер прозы — страховка, не разрешение

`_sanitize_prose` автоматически чистит длинные/средние тире в любом пользовательском тексте. **Не полагайся на него**: текст без тире «с нуля» читается естественнее. Санитайзер исправляет твою ошибку, но факт ошибки сохраняется в виде менее естественной фразы.

Поведение санитайзера:
- ` — ` / ` – ` (между словами) → `, `
- одиночные `—`/`–` (например в диапазонах `1—5`) → `-`

Что **не** санируется:
- LaTeX-строка в `r.formula(latex, ...)` (иначе сломался бы синтаксис).
- Канонические префиксы рисунков/таблиц (там тире обязательно по ГОСТ).
- Заголовки структурных элементов вроде «Введение», «Заключение».

### Dropdown в raw python-docx

Если совсем-совсем нужно что-то нестандартное (альбомная ориентация одной страницы, кастомная нумерация подзаголовков, специальный колонтитул), доступ к нижнему уровню — через `r.doc` (это `docx.Document`). Используй экономно — большинство задач решаются через профиль или через комбинацию существующих методов.
