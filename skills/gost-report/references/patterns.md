# Структура контента по типу работы

Шаблоны вызовов API под типичные виды российских студенческих работ. Все примеры подразумевают, что `r = Report(TitleConfig(...), profile=...)` уже создан.

## Лабораторная работа

```python
r.toc()
r.h1("Введение")
r.text("Цель работы освоить ...")     # 1-3 абзаца, без длинных тире
r.h1("Выполнение работы")
r.h2("Задание 1")
r.task("Задание 1. Сформулируй задачу.")
r.code("команда или код")
r.text("Результат показывает ...")
r.figure("screenshot.png", "Скриншот результата")
# h2 + task + code + text [+ figure] на каждое задание
r.h1("Заключение")
r.numbered([
    "Команда X освоена.",
    "Опции Y и Z изучены.",
])
```

## ВКР / выпускная квалификационная работа

```python
TitleConfig(
    work_type="Выпускная квалификационная работа",
    teacher_label="Руководитель",   # вместо «Проверил»
    ...
)
```

Структура:
```python
r.toc()
r.h1("Введение")
r.text("Актуальность исследования ...")
r.text("Объект исследования ...")
r.text("Предмет исследования ...")
r.text("Цель работы ...")
r.text("Задачи работы:")
r.numbered(["Задача 1", "Задача 2", "Задача 3"])

r.h1("Глава 1. Теоретические основы")
r.h2("1.1 Обзор литературы")
r.text("...")

r.h1("Глава 2. Реализация")
r.h2("2.1 Архитектура")
r.text("...")

r.h1("Глава 3. Результаты")

r.h1("Заключение")
r.numbered([
    "Проанализирован ...",
    "Реализован ...",
    "Получены результаты ...",
])

r.h1("Список литературы")
r.numbered([
    "Иванов И.И. Название работы. М.: Издательство, 2025. 320 с.",
    # ... источники в порядке упоминания или алфавитном
])
```

## Курсовая работа / курсовой проект

То же, что ВКР, но обычно короче (1-2 главы), `teacher_label="Руководитель"`, и без отдельных подразделов «объект/предмет», если кафедра не требует.

## Отчёт по практике

```python
TitleConfig(work_type="Отчёт по практике", ...)
```

Типичная структура:
```python
r.toc()
r.h1("Введение")
r.text("Период практики ...")
r.text("Место прохождения практики ...")
r.text("Цель практики ...")

r.h1("Описание организации")
r.text("Краткое описание компании / отдела / задач, выполняемых командой.")

r.h1("Выполненные работы")
r.h2("Задача 1. ...")
r.text("Что делал, какие технологии использовал, что получилось.")
r.h2("Задача 2. ...")
r.text("...")

r.h1("Заключение")
r.text("Подведение итогов, чему научился.")
```

## Домашнее задание / РГР

Самый простой случай — обычно как лабораторная без раздела «Введение»:
```python
r.toc()
r.h1("Условие задачи")
r.text("...")
r.h1("Решение")
r.formula(r"...")  # если есть формулы
r.text("...")
r.h1("Ответ")
r.text("...")
```

## Общие принципы

- **Введение и Заключение** — обязательные для любой работы (кроме коротких ДЗ).
- **Список литературы** — обязателен для ВКР/курсовых/больших отчётов.
- **«Введение», «Заключение», «Список литературы»** — это канонические заголовки, не выдумывай альтернативы.
- **Главы в ВКР** — пиши «Глава 1. ...», «Глава 2. ...» (с точкой и пробелом).
- **Задания в лабе** — пиши «Задание 1. ...», «Задание 2. ...» через `r.task(...)`.
- **Каждый `r.numbered(...)` стартует нумерацию заново** — если нужен один длинный список, передавай все элементы одним вызовом.
