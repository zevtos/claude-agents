# Чек-лист самопроверки отчёта по ГОСТ 7.32

Финальная проверка сгенерированного `.docx` перед сдачей. Большая часть
пунктов уже обеспечена библиотекой `gost_report` через активный
`UniversityProfile`; здесь — то, что требует человеческой проверки.

## Автоматически обеспечивает библиотека

Эти пункты **не требуют ручной проверки** — они зашиты в код и в активный профиль:

- Формат A4
- Шрифт Times New Roman, кегль 14, чёрный
- Межстрочный интервал 1,5
- Поля титульника и основного текста — согласно профилю (для `GOST_PROFILE`: 30/15/20/20 мм; для `ITMO_PROFILE`: титульник 30/20/10/10, тело 30/10/20/20)
- Абзацный отступ 1,25 см (в диапазоне 1–1,5 — норма ГОСТ)
- Выравнивание основного текста по ширине
- Нумерация страниц: внизу по центру, TNR 11, без точки, начиная со 2-й страницы
- На титульнике номер не отображается, но входит в общую нумерацию
- Заголовки: TNR полужирный, без подчёркивания, без точки в конце; кегли и выравнивание берутся из профиля
- H1 (структурные элементы и главы) — каждый с новой страницы (если `h1_new_page=True`), заглавными буквами (если `h1_uppercase=True`)
- Подпись рисунка снизу: «Рисунок N — Описание» (слово «Рисунок» — полностью)
- Подпись таблицы сверху: «Таблица N — Описание»
- Нумерация рисунков и таблиц — сквозная, автоматическая
- Нумерованные списки начинают счёт с 1 в каждом новом списке
- Город и год на титульнике без точки
- Город и год прижаты к нижнему полю плавающей borderless-таблицей — остаются на 1-й странице независимо от длины шапки/темы/ФИО/полей профиля
- На титульнике тема без кавычек (если вы не обернули её сами)
- Заголовок оглавления — `profile.toc_title` («ОГЛАВЛЕНИЕ» в ИТМО, «СОДЕРЖАНИЕ» в чистом ГОСТ)

## Требует человеческой проверки

### Профиль соответствует вашему вузу
- [ ] Используется правильный профиль: `ITMO_PROFILE` для ИТМО, иначе `GOST_PROFILE` или собственный `UniversityProfile`
- [ ] У вашего вуза действительно ГОСТ-овские поля 30/15/20/20 — если нет, переопределите `body_margin_*`
- [ ] Заголовок оглавления соответствует методичке вашей кафедры

### Содержание
- [ ] Тема в `TitleConfig.topic` написана **без** кавычек «»
- [ ] ФИО студента — в формате «Фамилия И.О.» (с инициалами через точку)
- [ ] Группа — в верном формате для вашего вуза
- [ ] Учёная степень преподавателя — корректно сокращена (к.т.н., д.ф.-м.н., к.э.н. и т.п.)
- [ ] Год — без буквы «г.» (только цифры)
- [ ] Для ВКР/курсовых — `teacher_label="Руководитель"`, не «Проверил»
- [ ] **В файле нет настоящих имён из примеров skill'а** — все ФИО подставлены пользователем

### Структура
- [ ] Есть введение, основные разделы, заключение
- [ ] Заключение — только результаты, без таблиц/графиков/ссылок
- [ ] Для лабораторных: каждое задание имеет постановку (`task()`) + команду (`code()`) + результат (текст или рисунок) + комментарий
- [ ] На все рисунки и таблицы в тексте есть ссылки
- [ ] Для ВКР/курсовых: есть список литературы (h1 «Список литературы»)

### После открытия в Word
- [ ] Оглавление обновлено: ПКМ по полю TOC → «Обновить поле» → «Обновить целиком»
- [ ] Заголовки в оглавлении совпадают с заголовками в тексте
- [ ] Все рисунки видны (не отображается крестик «битое изображение»)
- [ ] Таблицы не выходят за поля

### Если есть рисунки
- [ ] Файлы изображений лежат рядом со скриптом (или указан абсолютный путь)
- [ ] Изображения не слишком большие — при необходимости задайте `width_cm=10–15`
- [ ] Каждый рисунок имеет осмысленное описание в caption (не просто «Рисунок»)

## Быстрая проверка через PDF

Если есть LibreOffice, конвертируйте и просмотрите:

```bash
soffice --headless --convert-to pdf report.docx
```

Откройте PDF и пролистайте — должна быть видна:
1. Титульная страница (без номера внизу)
2. Страница оглавления (с номером 2)
3. Введение со страницы 3
4. Каждый h1 — на новой странице (если профиль это требует)
